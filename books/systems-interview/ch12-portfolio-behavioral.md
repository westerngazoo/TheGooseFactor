# Chapter 12: Portfolio & Behavioral for Systems Roles

What to know:
- Choose artifacts that demonstrate depth and reliability thinking

Examples of portfolio pieces:
- Driver with DMA + ISR handoff + backpressure + tests
- RTOS task set with deadlines and watchdog recovery demo
- Protocol parser with fuzzer and CRC/versioning tests

Lab:
- Create a one-page README per project with scope, how to run, and a 60-second video/GIF
- Write 5 story-bank bullets (context → action → result) with metrics

> :weightliftingoose: Your portfolio is your training log. Every lab you completed in this book is a potential portfolio piece. The 60-second README exercise forces you to distill: what did I build, why does it matter, and what did I measure? If you can't explain it in 60 seconds, you can't explain it in an interview.
>
> :happygoose: Story-bank bullets with metrics are *unforgeable*. "Reduced ISR latency from 3.2ms to 0.6ms" is concrete. "Improved performance" is noise. Numbers show you measured. Measurement shows you're rigorous. Rigor gets offers.
>
> :sharpgoose: "Driver with DMA + ISR handoff + backpressure + tests" — notice each piece adds one layer of sophistication. The driver alone is junior. Adding DMA shows hardware awareness. ISR handoff shows concurrency thinking. Backpressure shows systems design. Tests show professionalism. Stack all five.
