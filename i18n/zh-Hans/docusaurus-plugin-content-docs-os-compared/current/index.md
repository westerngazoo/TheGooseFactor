# 操作系统对比:Linux、seL4 与 GooseOS

> 三个操作系统。三种哲学。同样的硬件。操作系统所做的每一个设计决定 — 从"内核在内存中住在哪里?"到"两个进程如何交流?" — 都被不同目标的不同团队以不同方式回答过。这本书把这些答案并排放在一起。

## 这是什么?

这本书把操作系统设计中的每个基础话题拿出来,检视三个真实系统是如何处理它的:

- **Linux** — 三千万行的单内核巨人。在一切东西上运行,做一切事情,对每个问题都有一个解决方案(以及三个已弃用的替代方案)。
- **seL4** — 一万行的形式化验证微内核。数学上证明正确。极致的极简主义者。
- **GooseOS** — 用 Rust 写在 RISC-V 上的 1,500 行教学微内核。小到可以完全理解,真到可以在硬件上运行。

目标不是宣布赢家。是为了显示操作系统设计是一个**权衡空间**,而不是一组正确答案。Linux 为兼容性和性能优化。seL4 为安全性和形式化正确性优化。GooseOS 为清晰性和学习优化。每一个都为它的目标做出了"正确"的选择 — 而这些选择往往完全不同。

## 这本书写给谁?

- **计算机科学学生**,在教科书上学了操作系统理论,但想看真实系统如何实现
- **嵌入式开发者**,在 Linux、RTOS 和定制方案之间选择
- **GooseOS 读者**,想了解我们为什么这样设计的背景
- **好奇的工程师**,每天用 Linux 但从未看过内核内部
- **任何人**,曾想过"为什么 Linux 用*那种*方式做?"

## 如何阅读本书

每一章涵盖一个操作系统话题 — 启动、内存、IPC、调度等。在每一章里:

1. **问题** — 每个操作系统都必须解决什么
2. **Linux 的答案** — 单内核方式,带代码引用
3. **seL4 的答案** — 微内核方式,带形式化属性
4. **GooseOS 的答案** — 教学方式,带完整源代码
5. **对比表** — 决定的并排总结
6. **权衡分析** — 为什么每个选择在其背景下都讲得通

你可以从头到尾读,或跳到任何感兴趣的话题。每一章都是自包含的。

## 角色阵容

### Linux(1991–至今)

由 Linus Torvalds 创建,作为"只是一个爱好,不会像 GNU 那么大那么专业。"现在从手表到超级计算机上都在运行。内核是**单内核** — 设备驱动、文件系统、网络和调度都在内核空间运行,拥有对硬件的完全访问。当一个驱动出问题,整个内核都可能崩溃。

**关键指标**:约 30,000,000 行代码。约 400 个系统调用。约 30 年的向后兼容约束。

### seL4(2009–至今)

在澳大利亚的 NICTA(现 Data61/CSIRO)创建。一个有非凡属性的**微内核**:它有一个机器检查的数学证明,证明 C 代码正确实现了抽象规范。没有缓冲区溢出。没有空指针解引用。没有未定义行为。证明覆盖了内核 — 大约 10,000 行 C。

**关键指标**:约 10,000 行 C。约 3 个系统调用(Send、Receive、Yield + 变种)。形式化验证。

### GooseOS(2025–至今)

作为学习项目创建并实时记录。一个用 Rust 写的 RISC-V 微内核,目标是 WASM 原生的容器工作负载。目前有 10 个系统调用,在 VisionFive 2 板上运行。

**关键指标**:约 1,500 行 Rust。计划 18 个系统调用。在真实硬件上运行。

## 预览:系统调用接口

为了让你尝一尝每一章的样子,这里是系统调用对比的偷窥:

```
                Linux              seL4              GooseOS
                ─────              ────              ───────
系统调用数      ~450               ~3 (+ 变种)       18 (计划中)
约定            a7=编号            使用 IPC           a7=编号
分派            表查找             switch 语句        Rust match
入口            ecall → trap       ecall → trap      ecall → trap
出口            sret               sret              sret
```

相同的硬件机制(ecall/sret),关于内核到达那里后该做什么的截然不同的哲学。
